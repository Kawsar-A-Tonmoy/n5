# N5 Mastery

**N5 Mastery** is a Japanese vocabulary and kanji learning application currently focused on JLPT N5 content. It is available as both a web application and an Android app, making it easy to study wherever you are.

Built primarily with AI assistance and refined through continuous iteration, this project serves as both a practical learning tool and an experiment in AI-assisted software development.

## 🌐 Web Version

Use the web version instantly—no installation required.

**https://kawsar-a-tonmoy.github.io/n5/index.html**

## 📱 Android App

An Android version is also available for users who prefer studying on their phones. Download the latest APK from the repository's **Releases** section.

---

## Features

* Japanese vocabulary learning
* N5 kanji learning
* Hiragana and Katakana practice
* Interactive quizzes
* Reading practice
* Local progress saving
* Responsive interface
* Works both online (Web) and offline (Android)

> **Current Status:** The application currently focuses on JLPT N5 material. Additional vocabulary levels, kanji, grammar, and learning content are planned for future updates.

---

## About This Project

N5 Mastery started as an experiment to explore how far AI-assisted development could be pushed in building a real-world application.

While most of the code was generated with AI, the project has gone through extensive testing, debugging, and manual refinement to become a usable learning platform. The result isn't perfect—but it works, and it continues to improve.

If you browse the source code, you'll probably notice:

* Some unconventional coding patterns
* Architecture that evolved through rapid iteration
* A few questionable implementation choices
* More `div` elements than anyone would intentionally write

This repository prioritizes building useful features over showcasing perfect software engineering.

---

## Current Limitations

* Content is currently limited to N5 and is still incomplete.
* Some quizzes rely on straightforward answer matching.
* Parts of the codebase would benefit from further refactoring.
* Bugs and rough edges may still exist.
